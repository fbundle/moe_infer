# Flash-MoE

Pure C/Metal inference engine for Mixture-of-Experts models on Apple Silicon. Streams expert weights from SSD on demand — no Python, no frameworks, just C, Objective-C, and hand-tuned Metal shaders.

Currently running **Qwen3.5-35B-A3B** (35B params, 37GB 4-bit) at ~5.2 tok/s on Apple M4 with 16GB RAM.

## Building from Scratch

Start from the HuggingFace model and end with a running inference engine.

### 1. Download the pre-quantized model

```bash
# Install huggingface_hub CLI if needed
pip install huggingface_hub

# Download the MLX 4-bit model (~19GB)
huggingface-cli download mlx-community/Qwen3.5-35B-A3B-4bit \
  --local-dir hub/models--mlx-community--Qwen3.5-35B-A3B-4bit
```

### 2. Generate model config

```bash
python helpers/gen_model_config.py \
  hub/models--mlx-community--Qwen3.5-35B-A3B-4bit/config.json
```

This creates `src/model_config.h` with all dimension constants, expert layout offsets, and special tokens for your model.

### 3. Extract non-expert weights

```bash
python helpers/extract_weights.py
```

Writes `data/model_weights.bin` (~1.38 GB) and `data/model_weights.json` manifest. These are mmap'd at runtime.

### 4. Repack expert weights

```bash
python helpers/repack_experts_4bit.py
```

Extracts per-layer binary files into `packed_experts/layer_00.bin` through `layer_39.bin` (~453 MB each, 18.1 GB total). These are streamed from SSD on demand.

### 5. Generate experiment config

```bash
python helpers/gen_config.py --k 8 --cache-mode 0 --gpu-linear 1 --prediction 0
```

Creates `src/config.h` with compile-time experiment knobs. Run `make autotune` later to find the best values.

### 6. Build

```bash
make
```

Builds `bin/infer` (inference engine) and `bin/chat` (interactive TUI).

### 7. Run

```bash
# Basic inference
./bin/infer --prompt "Explain quantum computing" --tokens 100

# Interactive chat
./bin/chat --k 8

# Per-layer timing breakdown
./bin/infer --prompt "Hello" --tokens 20 --timing
```

```
bin/                   # Build output (gitignored)
```

## Experiment System

All experiment knobs are **compile-time** — no runtime overhead from conditionals.

`src/config.h` (generated by `helpers/gen_config.py`):

| Define | Values | Description |
|--------|--------|-------------|
| `ACTIVE_EXPERTS_PER_LAYER` | 2, 4, 8 | Active experts per layer |
| `CACHE_MODE` | 0, 1 | 0 = OS page cache, 1 = malloc cache |
| `CACHE_ENTRIES` | 0..N | Malloc cache entries (mode 1 only) |
| `GPU_LINEAR` | 0, 1 | 1 = fused GPU delta-net, 0 = CPU BLAS |
| `PREDICTION` | 0, 1 | Temporal expert prediction (paper: -18%) |

```bash
# Manual: set config, rebuild, run
python helpers/gen_config.py --k 4
make && ./bin/infer --prompt "Hello" --tokens 20

# Automatic: sweep all configs, find the best
make autotune
```

## Architecture

35B-A3B model: 40 layers (30 linear attention + 10 full attention), 256 experts, K=8 active. Hidden dim 2048, head dim 256.

### Key Techniques

1. **SSD Expert Streaming** — Expert weights (37GB 4-bit) read from NVMe SSD on demand via parallel `pread()`. Only K active experts per layer are loaded (~1.77MB each). The OS page cache handles caching — no custom cache needed.

2. **FMA-Optimized Dequant Kernel** — Rearranges `(nibble * scale + bias) * x` to `fma(nibble, scale*x, bias*x)`, letting the GPU fused multiply-add unit do dequant+multiply in one instruction. 12% faster than naive formulation.

3. **Metal Compute Shaders** — 4-bit dequantized matvec, fused SwiGLU, RMS norm, batched attention, GPU RoPE, MoE combine + residual.

4. **Accelerate BLAS for Linear Attention** — GatedDeltaNet recurrence uses `cblas_sgemv` for the 64-head state matrix update. 64% faster than scalar code.

5. **Trust the OS** — The OS page cache achieves ~71% hit rate naturally. Every custom caching approach (Metal LRU, malloc cache) was slower due to GPU memory pressure.

## What We Tried

| Approach | Result | Impact |
|----------|--------|--------|
| FMA dequant kernel | GPU compute -12% | +12% tok/s |
| Trust OS page cache | Deleted Metal LRU | +38% |
| BLAS delta-net (Accelerate) | cpu_attn 0.78→0.28ms | +64% attn |
| Deferred GPU CMD3 | CPU/GPU overlap | Pipeline |
| Fused GPU combine+norm | Eliminates CPU round-trip | Pipeline |
| LZ4 expert compression | -13% | Decompress > savings |
| Temporal expert prediction | -18% | 25% hit rate |
| MTP speculative decoding | Break-even | MoE I/O scales per-token |
