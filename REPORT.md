# Port Report: Flash-MoE → Qwen3.5-35B-A3B

## Summary

Ported the Flash-MoE Metal inference engine from Qwen3.5-397B-A17B (397B params, 209GB) to Qwen/Qwen3.5-35B-A3B (35B params, ~37GB 4-bit). The engine is now model-agnostic via compile-time `#define` constants in `model_config.h`. Used the pre-quantized `mlx-community/Qwen3.5-35B-A3B-4bit` model from HuggingFace.

**Result**: Inference runs at ~5.2 tok/s (K=8) on Apple M4 with 16GB RAM. Output is coherent and factually correct.

| Model size (4-bit) | 209GB | 37GB |

## Files Created

### `metal_infer/model_config.h`
Compile-time model constants. All dimension-dependent code uses `#define` constants from here.
Generated by `gen_model_config.py` from the HuggingFace `config.json`.

### `metal_infer/gen_model_config.py`
Reads a HuggingFace `config.json` (handles nested `text_config` for multimodal models) and generates `model_config.h` with all dimension constants, expert layout offsets, and special tokens.

### `metal_infer/repack_experts_4bit.py`
Extracts already-quantized 4-bit expert weights from the MLX pre-quantized model's 3D tensor format (`[num_experts, out_dim, packed_in_dim]`) and writes per-layer binary files (`packed_experts/layer_XX.bin`).

The MLX model stores expert weights as 3D tensors:
- `switch_mlp.gate_proj.weight`: `[256, 512, 256]` U32
- `switch_mlp.down_proj.weight`: `[256, 2048, 64]` U32
- Scales/biases: `[256, out_dim, out_dim/group_size]` BF16

The script slices per-expert and writes them in the order the C engine expects:
`gate_w → gate_s → gate_b → up_w → up_s → up_b → down_w → down_s → down_b`

Output: 40 files × 453 MB = 18.1 GB total.

## Files Modified

### `metal_infer/extract_weights.py`
Updated tensor name sanitization for MLX naming convention:
- `language_model.model.layers.N.X` → `model.layers.N.X`
- `language_model.lm_head.X` → `lm_head.X`
- `language_model.model.embed_tokens.X` → `model.embed_tokens.X`
- `language_model.model.norm.X` → `model.norm.X`
- Also handles raw Qwen3.5 format: `model.language_model.X` → `model.X`

Expert pattern regex already covered both raw (`mlp.experts.*`) and MLX (`switch_mlp.*`) naming.

### `metal_infer/infer.m` — Critical Bug Fix
**Hardcoded expert buffer offsets from the 397B model caused NaN output.** The GPU expert encoding functions (`gpu_encode_experts_batched`, `gpu_encode_expert_forward`, `gpu_encode_expert_forward_slot`, `gpu_encode_expert_forward_slot_buf`) and CPU fallback paths used hardcoded offsets like `gate_s_off = 2097152` that exceeded the 35B model's `EXPERT_SIZE = 1,769,472`, causing out-of-bounds reads.

All 8 occurrences replaced with `#define` constants from `model_config.h`:
```c
// Was (397B hardcoded):
gate_s_off = 2097152;  gate_b_off = 2228224;
up_w_off   = 2359296;  up_s_off   = 4456448;  up_b_off   = 4587520;
down_w_off = 4718592;  down_s_off = 6815744;  down_b_off = 6946816;

// Now (model-config-driven):
gate_w_off = GATE_W_OFF; gate_s_off = GATE_S_OFF; gate_b_off = GATE_B_OFF;
up_w_off   = UP_W_OFF;   up_s_off   = UP_S_OFF;   up_b_off   = UP_B_OFF;
down_w_off = DOWN_W_OFF; down_s_off = DOWN_S_OFF; down_b_off = DOWN_B_OFF;
```

## Model Download

Pre-quantized 4-bit model downloaded from HuggingFace:
`mlx-community/Qwen3.5-35B-A3B-4bit` (~19GB, 4 safetensors shards)

Located at: `hub/models--mlx-community--Qwen3.5-35B-A3B-4bit/`

## Weight Extraction Pipeline

1. **Non-expert weights**: `extract_weights.py` → `model_weights.bin` (1.38 GB, 1397 tensors) + `model_weights.json` manifest
2. **Expert weights**: `repack_experts_4bit.py` → `packed_experts/layer_00.bin` through `layer_39.bin` (453 MB each, 18.1 GB total)

Both use the MLX pre-quantized format directly — no manual quantization needed. The MLX model already has weights in U32 packed 4-bit format with BF16 scales/biases.

## Configuration Generation

`gen_model_config.py` reads `config.json` and computes:
- All dimension constants
- 4-bit expert layout offsets (`GATE_W_OFF`, `GATE_S_OFF`, etc.)
- 2-bit expert layout offsets
- Special tokens (EOS, think tags)
- Default model path

Run after downloading a new model to generate its `model_config.h`.

## Known Issues

- **K=4 quality degradation**: The 35B model was trained with 8 experts per token. Using K=4 produces degraded output (garbage tokens after a few steps). Default is now K=8 (= NUM_EXPERTS_PER_TOK).
- **2-bit not tested**: The 2-bit expert path has layout constants in `model_config.h` but no 2-bit expert files have been generated for the 35B model.

## Lessons

1. **Never hardcode model-dependent constants in C**. The expert buffer offsets were the root cause of a multi-hour debugging session. All such values should be `#define`d in a config header.
2. **Pre-quantized models save massive time**. Using `mlx-community/Qwen3.5-35B-A3B-4bit` eliminated the need for manual quantization (~400 matmul tensors) and avoided an entire class of quantization bugs.
3. **Weight naming is the interface contract**. MLX's `sanitize()` function in `qwen3_5_moe.py` defines the naming convention. Matching it exactly in `extract_weights.py` prevented lookup failures.
