# Port Report: Flash-MoE → Qwen3.5-35B-A3B

## Summary

Ported the Flash-MoE Metal inference engine from Qwen3.5-397B-A17B (397B params, 209GB) to Qwen/Qwen3.5-35B-A3B (35B params, 37GB 4-bit). The engine is now model-agnostic via compile-time `#define` constants in `model_config.h`. Used the pre-quantized `mlx-community/Qwen3.5-35B-A3B-4bit` model from HuggingFace.

**Result**: ~5.2 tok/s (K=8) on Apple M4 with 16GB RAM.

| Model size (4-bit) | 209GB | 37GB |

## Files Created

### `src/model_config.h`
Compile-time model constants generated by `gen_model_config.py` from the HuggingFace `config.json`. All dimension-dependent code uses these defines.

### `helpers/gen_model_config.py`
Reads a HuggingFace `config.json` and generates `src/model_config.h` with dimension constants, expert layout offsets, and special tokens.

### `helpers/repack_experts_4bit.py`
Extracts MLX pre-quantized 4-bit expert weights from 3D tensors `[num_experts, out_dim, packed_in_dim]` and writes per-layer binary files (`packed_experts/layer_XX.bin`). Output: 40 files × 453 MB = 18.1 GB.

### `src/config.h` + `helpers/gen_config.py`
Compile-time experiment configuration system. `config.h` contains mandatory defines (`ACTIVE_EXPERTS_PER_LAYER`, `CACHE_MODE`, `GPU_LINEAR`, `PREDICTION`) with no `#ifndef` guards — the engine won't compile without it. `gen_config.py` generates the header programmatically.

### `autotune.py`
Sweeps the experiment parameter space (K ∈ {2,4,8}, GPU_LINEAR ∈ {0,1}, PREDICTION ∈ {0,1}), builds each config, benchmarks tok/s, captures output text for quality assessment, and reports the optimal 4-bit configuration.

### `Makefile`
Clean root-level Makefile. Builds `src/` into `bin/`. Experiment configs handled by `autotune.py` — no per-config Makefile targets needed.

### `README.md`
Tool documentation covering quick start, project structure, experiment system, and architecture.

## Files Modified

### `src/infer.m`
- **Hardcoded offset fix**: 8 occurrences of 397B expert buffer offsets replaced with `model_config.h` defines. These were the root cause of NaN output.
- **Compile-time experiment system**: Includes `config.h`, uses defines for K, cache mode, GPU linear, prediction.
- **Removed runtime flags**: `--k`, `--cache-entries`, `--malloc-cache`, `--cpu-linear`, `--gpu-linear`, `--predict` — all now compile-time via `config.h`.
- **Stack arrays**: `layer_fds`, `layer_mmaps`, `layer_mmap_sizes` converted from `calloc` to stack (NUM_LAYERS is compile-time).
- Default K changed from 4 to 8 (= NUM_EXPERTS_PER_TOK).

### `helpers/extract_weights.py`
Updated tensor name sanitization for MLX naming convention:
- `language_model.model.layers.N.X` → `model.layers.N.X`
- `language_model.lm_head.X` → `lm_head.X`
- `language_model.model.embed_tokens.X` → `model.embed_tokens.X`

### `helpers/pack_experts.py`
Added `tqdm` progress bars with per-layer timing.

## Weight Extraction Pipeline

1. **Non-expert weights**: `extract_weights.py` → `data/model_weights.bin` (1.38 GB) + `data/model_weights.json` manifest
2. **Expert weights**: `repack_experts_4bit.py` → `packed_experts/layer_00.bin` through `layer_39.bin` (453 MB each)

Both use MLX pre-quantized format — no manual quantization needed.

## Known Issues

- **K=4 quality degradation**: Model trained with 8 experts/token. K=4 produces degraded output. Default is now K=8.
- **2-bit not tested**: Layout constants exist in `model_config.h` but no 2-bit expert files generated.
- **Malloc cache on 16GB**: ~2581 entries needed for 80% hit rate requires ~17GB, tight on 16GB machines.

## Lessons

1. **Never hardcode model-dependent constants in C**. The expert buffer offsets caused a multi-hour debugging session. All such values should be `#define`d in a config header.
2. **Pre-quantized models save time**. MLX pre-quantized format eliminated manual quantization (~400 matmul tensors) and avoided an entire class of quantization bugs.
3. **Weight naming is the interface contract**. MLX's `sanitize()` function defines the naming convention — matching it exactly in `extract_weights.py` prevents lookup failures.
4. **Compile-time over runtime for experiments**. Moving K, cache mode, etc. to compile-time defines eliminates branching overhead and ensures fair benchmarking. The `autotune.py` approach (regenerate config.h → rebuild → benchmark) is clean and reproducible.
